# 프로젝트 이미지 스타일 가이드

> 버전: 2.0
> 작성일: 2026년 6월 30일  
> 적용 범위: 프로젝트에서 수집·편집·생성하여 사용하는 모든 이미지

## 1. 이미지 사용 원칙

이 프로젝트는 **실제 사진 자료 사용을 원칙**으로 한다. AI 생성 이미지는 적절한 사진을 확보할 수 없을 때만 보조 수단으로 사용한다.

우선순위:

1. 사용 권한과 출처가 확인된 실제 사진
2. 사용 권한이 확인된 사진을 프로젝트 형식에 맞게 편집한 이미지
3. 사진 자료가 없거나 장면을 설명하기 어려울 때 사용하는 AI 생성 이미지

AI 생성 이미지는 실제 경기의 증거 사진이나 보도사진처럼 제시하지 않는다. 실제 사건을 재구성한 경우 이미지 주변에 **`AI 생성 이미지`** 또는 **`상황 재구성 일러스트`**라고 표시한다.

## 2. 기본 방향

사진과 생성 이미지 모두 **현대적인 스포츠 에디토리얼** 분위기를 유지한다. 경기의 속도감과 현장감은 살리되, 자료조사 결과와 데이터가 명확하게 전달되는 차분한 시각 체계를 우선한다.

핵심 인상:

- 역동적이지만 과도하게 자극적이지 않음
- 전문적이고 신뢰할 수 있음
- 한국어 콘텐츠에 자연스럽게 어울림
- 모바일과 발표 자료에서 빠르게 이해할 수 있음
- 여러 이미지를 함께 배치해도 하나의 시리즈처럼 보임

## 3. 실제 사진 사용 기준

### 사진 선택

- 문서에서 설명하는 경기·선수·장소와 직접 관련된 사진을 우선한다.
- 촬영 시점과 맥락이 확인되지 않은 사진은 해당 경기의 사진처럼 사용하지 않는다.
- 핵심 피사체가 선명하고 작은 화면에서도 상황을 알아볼 수 있는 사진을 선택한다.
- 워터마크를 제거하거나 사진의 의미를 바꾸는 편집은 하지 않는다.
- 단순히 검색 결과에 노출된 사진이라는 이유만으로 사용하지 않는다.

### 사용 권한과 출처

- 공식 배포 자료, 직접 촬영한 사진, 라이선스가 명시된 사진을 우선한다.
- 사진마다 원문 URL, 제공자 또는 촬영자, 라이선스, 확인일을 기록한다.
- 출처 표기가 요구되는 사진은 이미지 캡션이나 문서 하단에 표시한다.
- 편집 허용 여부가 불분명한 사진은 원본 그대로 사용하거나 다른 자료를 찾는다.
- 사용 권한을 확인할 수 없으면 프로젝트에 포함하지 않고 AI 생성 이미지 사용 여부를 검토한다.

### 사진 편집

- 원본 파일은 변경하지 않고 별도로 보관한다.
- 자르기, 크기 조절, 밝기·대비·색온도의 가벼운 보정만 허용한다.
- 선수, 공, 경기 결과 등 사실관계를 바꾸는 합성이나 제거는 하지 않는다.
- 제목을 넣을 때는 반투명 딥 네이비 오버레이를 사용해 가독성을 확보한다.
- 여러 사진의 색감을 맞추더라도 팀 유니폼과 경기장 색상을 왜곡하지 않는다.

## 4. AI 생성 이미지 사용 기준

다음 조건 중 하나에 해당할 때만 생성 이미지를 사용한다.

- 필요한 장면의 사진이 존재하지 않거나 합법적으로 확보할 수 없음
- 전술, 데이터, 시간의 흐름처럼 사진 한 장으로 설명하기 어려움
- 실제 장면이 아니라 개념·예상·비유를 시각화하는 콘텐츠임
- 사용 가능한 사진이 해상도 또는 구도상 목적에 적합하지 않음

### 생성 스타일

- 실제 경기 사진과 혼동되지 않도록 **2D 스포츠 애니메이션 에디토리얼**을 기본으로 한다.
- 굵고 정돈된 외곽선, 제한된 셀 셰이딩, 단순화한 관중과 배경을 사용한다.
- 선수는 특정 실존 인물의 얼굴을 복제하지 않고 일반적인 인물로 표현한다.
- 실제 상황을 재구성할 때는 충돌, 공의 위치, 선수 방향 등 핵심 사실만 정확히 전달한다.
- 이미지 안에 글자나 가상의 로고를 생성하지 않고 문구는 후편집으로 추가한다.

생성 이미지에는 다음 캡션 중 하나를 사용한다.

```text
AI 생성 이미지
AI를 활용한 상황 재구성 일러스트
전술 이해를 돕기 위한 AI 생성 이미지
```

## 5. 공통 색상

| 용도 | 색상 | HEX |
|---|---|---|
| 주 배경·오버레이 | 딥 네이비 | `#071426` |
| 보조 배경 | 슬레이트 블루 | `#132A44` |
| 핵심 강조 | 에너지 레드 | `#E63946` |
| 데이터 강조 | 터프 그린 | `#20C997` |
| 본문·밝은 배경 | 오프화이트 | `#F5F7FA` |
| 보조 정보 | 쿨 그레이 | `#AAB4C3` |

사용 원칙:

- 실제 사진의 고유 색감은 존중하고 브랜드 색상은 오버레이와 그래픽에만 제한적으로 사용한다.
- 한 이미지에서 주조색 1개, 강조색 1~2개만 사용한다.
- 대한민국 관련 강조에는 레드와 네이비를 사용하되 국기 문양을 장식처럼 남용하지 않는다.
- 상대 팀의 국가색은 보조적으로만 사용한다.
- 텍스트와 배경은 충분한 명도 대비를 확보한다.

## 6. 타이포그래피

- 한국어: `Pretendard` 또는 `Noto Sans KR`
- 영문·숫자: `Montserrat` 또는 `Inter`
- 제목은 굵고 짧게, 본문은 보통 굵기로 사용한다.
- 한 이미지에 글꼴은 최대 2종, 굵기는 최대 3단계로 제한한다.
- AI 이미지 생성 단계에서는 글자를 넣지 않고 필요한 문구는 후편집으로 추가한다.

## 7. 구도

- 핵심 피사체는 중앙 또는 3분할 교차점에 배치한다.
- 제목이나 통계를 넣을 예정인 방향에 25~35%의 여백을 남긴다.
- 원본 사진을 자를 때 핵심 선수, 공, 골문 등 상황 이해에 필요한 요소를 보존한다.
- 대결 구도는 좌우 균형을 사용하되 모든 이미지를 정면 대칭으로 만들지는 않는다.
- 인포그래픽은 위에서 아래 또는 왼쪽에서 오른쪽으로 읽히도록 구성한다.
- 썸네일에서는 작은 크기로 보더라도 핵심 피사체와 대비가 유지되어야 한다.

## 8. 이미지 유형별 규격

| 용도 | 권장 비율 | 권장 크기 |
|---|---:|---:|
| 문서 대표 이미지 | 16:9 | 1600×900 |
| 발표 자료 | 16:9 | 1920×1080 |
| 카드 뉴스 | 1:1 | 1080×1080 |
| 세로형 콘텐츠 | 4:5 | 1080×1350 |
| 배너·헤더 | 3:1 | 1800×600 |
| 투명 배경 요소 | 자유 | 긴 변 1600px 이상 |

사진 원본은 제공된 형식으로 보존한다. 편집본은 고품질 JPG 또는 WebP, 투명 배경 그래픽과 생성 일러스트는 PNG 또는 WebP로 저장한다.

## 9. 사진이 없을 때의 생성 프롬프트

아래 구조를 기본 템플릿으로 사용한다.

```text
[이미지 목적], [주요 장면과 피사체],
2D sports animation editorial, clean bold outlines,
limited cel shading, dynamic but credible action,
deep navy base with restrained red and green accents,
clean composition, strong visual hierarchy,
[텍스트를 넣을 위치]에 넉넉한 negative space,
[화면 비율], clearly illustrated rather than a real photograph,
no text, no logo, no watermark
```

예시:

```text
사용 가능한 사진이 없는 대한민국과 멕시코의 국제 축구 경기 프리뷰용 이미지,
서로 마주 보는 두 팀의 축구 선수를 실제 인물이 아닌 일반적인 선수로 표현,
야간 경기장과 긴장감 있는 관중석,
2D sports animation editorial, clean bold outlines,
limited cel shading, dynamic but credible action,
deep navy base with restrained red and green accents,
왼쪽 상단에 제목을 넣을 넉넉한 negative space,
16:9, clearly illustrated rather than a real photograph,
no text, no official crest, no logo, no watermark
```

## 10. 피해야 할 요소

- 출처나 사용 권한이 확인되지 않은 사진
- 다른 경기의 사진을 해당 경기의 장면처럼 소개하는 것
- 사진의 워터마크나 저작자 표시를 제거하는 것
- 실제 사진과 AI 생성 이미지를 구분하지 않고 함께 사용하는 것
- AI 생성 이미지를 실제 보도사진처럼 설명하는 것
- 공식 FIFA·대회·협회·구단 로고를 허가 없이 재현하는 것
- 실제 선수와 혼동될 정도로 특정 인물의 얼굴을 생성하는 것
- 읽을 수 없는 AI 생성 문자, 가짜 후원사와 가짜 엠블럼
- 손가락, 팔다리, 축구공 크기 등 신체·사물 비율 오류
- 지나친 네온 색상, 불꽃, 번개, 렌즈 플레어
- 과도한 국기 사용과 국가에 대한 고정관념적 표현

## 11. 파일 관리 규칙

파일명은 영문 소문자와 하이픈을 사용한다.

```text
출처유형-주제-용도-버전.확장자
```

예시:

```text
photo-korea-mexico-match-v01.jpg
generated-korea-mexico-match-preview-v01.png
graphic-korea-mexico-match-stats-v01.webp
```

권장 하위 폴더:

```text
image/
├─ photo/
│  ├─ original/ # 출처와 사용 권한을 확인한 사진 원본
│  └─ edited/   # 프로젝트 규격에 맞게 편집한 사진
├─ generated/
│  ├─ draft/    # AI 생성 비교 시안
│  └─ final/    # 선택·검수된 AI 생성 이미지
├─ final/       # 실제 프로젝트에 사용하는 최종본
└─ reference/   # 프로젝트에 직접 사용하지 않는 스타일 참고 자료
```

같은 이미지의 수정본은 `v01`, `v02`처럼 버전을 올린다. 최종본에도 버전을 유지하고 `final-final` 같은 이름은 사용하지 않는다.

사진의 출처 정보에는 다음 항목을 기록한다.

```text
파일명:
원문 URL:
제공자/촬영자:
라이선스 또는 사용 근거:
확인일:
편집 내용:
```

## 12. 사용 전 확인표

- [ ] 먼저 사용할 수 있는 실제 사진을 찾아보았는가?
- [ ] 사진의 출처와 사용 권한을 확인했는가?
- [ ] 사진이 문서에서 설명하는 실제 장면과 일치하는가?
- [ ] 원본과 편집본을 구분해 보관했는가?
- [ ] 사진이 없어 AI 이미지를 사용해야 하는 이유가 명확한가?
- [ ] 생성 이미지에 AI 제작 또는 상황 재구성 표시를 준비했는가?
- [ ] 이미지의 용도와 출력 비율이 정해졌는가?
- [ ] 기존 이미지와 색상·구도가 일관적인가?
- [ ] 글자를 넣을 여백이 충분한가?
- [ ] 공식 로고와 상표를 무단으로 포함하지 않았는가?
- [ ] 인물과 축구 장면이 자연스러운가?
- [ ] 파일명이 관리 규칙을 따르는가?

## 13. 가이드 갱신

프로젝트의 최종 매체와 브랜드 방향이 정해지면 다음 항목을 보완한다.

- 공식 로고와 브랜드 색상
- 고정 타이포그래피
- 이미지별 실제 출력 규격
- 사용 가능한 공식 사진 제공처
- 프로젝트에 적용할 구체적인 라이선스 기준
- 사진 출처 기록 파일의 위치와 형식
